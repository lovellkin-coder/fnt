(가칭) 디지털존 F&T 사업 기획안

Fragmentation & Token · 전략기획팀 · 2026.07.06 · Draft v2

01사업 정의

중소·중견기업을 위한 맞춤형 통합 비즈니스 솔루션 플랫폼

  • 각기 다른 공급사의 SW 솔루션(그룹웨어·ERP·CRM·HR 등)을 고객 맞춤형으로 큐레이션하여 디지털존에서 판매·제공
  • 고객이 선택한 솔루션 간 백엔드 데이터 자동 연동 — 별도 SI 구축 없이 구매 즉시 사용 가능
  • 복수의 솔루션을 이용하더라도 디지털존 단일 청구 체계로 통합 정산
  • 사용량·현황은 디지털존 대시보드에서 통합 확인
02시장 배경 및 Pain Point

중소·중견기업은 비용·기술적 한계로 SW 솔루션의 통합 운영에 어려움을 겪고 있으며, 이를 해소할 시장 수요가 존재함

Pain Point 01
SaaS 파편화 · 데이터 단절
  • ERP·CRM·HR 등 솔루션을 각각 다른 공급사에서 도입, 시스템 간 데이터 연동 없이 운영
  • 하나의 데이터 변경 시 각 시스템에 별도 반영 필요 — 운영 비효율 발생
  • 통합 현황 파악이 어려워 의사결정 속도 저하
Pain Point 02
통합 개발·유지보수 부담
  • 자체 통합 시스템 구축 시 SI 개발 비용 부담 — SMB는 감당 어려움
  • SI 외주 방식은 개발자 퇴사·교체 시 히스토리 단절, 고도화 난이도 상승
  • 기술 내재화도 채용·판관비 부담으로 쉽지 않은 선택
Pain Point 03
복수 정산 피로도
  • 솔루션별 공급사와 개별 계약·세금계산서 발행 관리 필요
  • 담당자 부재·교체 시 정산 이력 파악 어려움
  • 갱신 시기·조건이 솔루션마다 달라 누락 리스크 상존

핵심 가치
솔루션 큐레이션
  • 분야별 검증된 SW 솔루션을 디지털존에서 제공
  • 고객은 필요한 솔루션만 선택하여 구독
데이터 자동 연동
  • 선택한 솔루션 간 데이터를 디지털존 서버가 중간에서 연결
  • 별도 SI 없이 구매 즉시 연동된 형태로 사용 가능
  • 디지털존에서 구매한 솔루션에 한해 연동 제공 (별도 계약 중인 SW는 해지 후 이전하거나 기존 그대로 유지하는 방식 중 선택 가능)
통합 빌링 · 관리
  • 복수 솔루션 이용료를 디지털존이 일괄 수령하여 각 벤더사에 정산
  • 고객은 매월 1회 단일 청구서로 정산 완결
  • 사용량·현황은 디지털존 대시보드에서 통합 확인
04기술 구조

벤더사 API를 디지털존 서버에서 수집·저장하고, 고객별 선택 솔루션에 맞게 데이터를 재분배하는 미들레이어 구조

🖥️ 온프레미스 — 3가지 방안 검토
1안 · 에이전트 설치

고객사 서버에 에이전트를 심어 자동 집계

  • 사용량을 실시간으로 자동 수집 — 종량제 적용 가능
  • 단, 온프레미스 도입 업체는 보수적 성향이 강해 에이전트 설치 수용 가능성 낮음
  • 현실적으로 적용 난이도 높음
2안 · 월 정액제

사용량 관계없이 월 고정 금액 청구

  • 별도 집계 없이 운영 가능 — 구조 단순
  • 고객 입장에서 비용 예측 용이
  • 실사용량과 무관하게 동일 과금 — 헤비 유저 대응 어려움
3안 · 고객 직접 미터링

고객이 사용량 직접 업로드 + 연 1회 현장 검수

  • 고객이 월별 사용량을 직접 업로드하여 청구
  • 디지털존은 연 1회 현장 방문 검수 — 오차 발생 시 추가 과금
  • 에이전트 불필요 — 온프레미스 업체 수용 가능성 상대적으로 높음

※ 온프레미스 과금 방안은 벤더사 협의 및 고객 수용성 검토 후 확정 예정

07필요 인력 / R&R

본 사업 추진을 위한 개발 직군별 주요 역할 정의

PM

프로젝트 총괄

  • 전체 일정·이슈 관리
  • 벤더사·디지털존 연동 조율
  • 토큰 과금·데이터 연동 요건 정의
  • 이해관계자 커뮤니케이션
Backend

서버·API·과금

  • 벤더사 API 연동 및 데이터 수집
  • 데이터 라우팅 로직 구현
  • API 콜수 기반 토큰 과금 엔진
  • 정산·페이먼트 연동
Frontend

화면·UX

  • 솔루션 선택·구매 UI
  • 사용량 통합 대시보드
  • 정산서·페이먼트 페이지
  • 디지털존 내 연동 화면
  • 위 개발 인력 외 서비스 기획, QA, 퍼블리셔 등 비개발 직군은 세부 요건 정의 후 별도 산정
  • 인력 조달 방식(내재화·외주)은 IT 부문 협의 후 결정
08기대효과 및 비전

사업을 통해 기대되는 고객·플랫폼·전략적 효과

VISION
SMB의 디지털 전환을 리드하는 B2B SaaS 통합 플랫폼
구매·연동·관리·정산까지 디지털존 하나로 완결
고객 관점
  • SaaS 도입·운영 비용 절감
  • 데이터 연동으로 운영 효율 향상
  • 정산 업무 단순화 및 누락 리스크 제거
디지털존 관점
  • 단순 커머스를 넘어 구독·관리 기반 플랫폼으로 진화
  • 고객·파트너·벤더사 생태계를 디지털존 안에서 구축
  • 마켓플레이스 확장의 실질적 발판 마련
전략적 의미
  • 토큰 과금 체계를 선제적으로 운영 — 제도 변화 대응 역량 확보
  • 데이터 기반 과금·운영 노하우 내재화
  • 향후 솔루션 카테고리 확대 시 확장성 확보
